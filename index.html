Git/CLI

The worst part about programming is finding out that you wake up one day and realise that your code doesn't work! Or even worse... your computer spontanesouly self combusted! In either case you need to find a working copy of your code stat.
Welcome, GIT.


Git works from the command line (or application iirc) and allows you to store your work locally and on the cloud. It helps programmers implement version control for their projects and lets multiple users work independently, without the headache of untangling where Bob's code ends and Sally's begins.


Here are some example commands and their explanations

git init - Initizialises the directory as a Git repo, i.e. Git will now pay attention to what happens in this directory.

git status - Tells us what branch we are currently working on, the status of our commits on the branch and what files are being watched or have changes.

git add - Use this when you want to add a file to your next commit. This will make sure that git looks for changes to this file across your commits.

git commit - This will create some meta data to associate the added files together. It's useful to include your own message with the -m flag!

git push - Send your last commit to your online repository, this will send any changes/commits from your local copy to your remote copy where you can see a more macroscopic look on the history of your work!
